QDB » Persistent message queues with replay

  • REST API for easy access from curl and any programming language
  • Use on its own or as a front end to RabbitMQ
  • Replay messages by timestamp or id
  • Low memory footprint (run on an EC2 micro instance!)

The following example creates a database and queue and configures it to output to RabbitMQ:

{% highlight bash %} $ curl -XPOST http://127.0.0.1:9554/db/acme $ curl -XPOST http://127.0.0.1:9554/db/acme/q/incoming $ curl -XPOST http://127.0.0.1:9554/db/acme/q/incoming/out/rabbit \ -d type=rabbitmq -d url=amqp://127.0.0.1 -d exchange=incoming # these operations are idempotent i.e. if the object already exists it is updated {% endhighlight %}

Its easy to POST messages to the queue:

{% highlight bash %} $ curl -XPOST http://127.0.0.1:9554/db/acme/q/incoming/messages?routingKey=abc -d @message.json # the contents of message.json are now in RabbitMQ {% endhighlight %}

And stream them out again or redeliver:

{% highlight bash %} $ curl http://127.0.0.1:9554/db/acme/q/incoming/messages?at=10:13 # messages since 10:13 today are streamed to the console $ curl -XPOST http://127.0.0.1:9554/db/acme/q/incoming/out/rabbit -d at=2013-06-24 # messages since 24 June 2013 are redelivered to RabbitMQ {% endhighlight %}

QDB is written in Java and runs on any system with a Java 6 or 7 VM. It is open source and Apache licensed.

Installation

Download a QBD distribution, unpack it somewhere and run start.sh.
